# 项目部署指南

## 🚀 推荐方案：Render.com 部署（最简单）

### 1. 注册 Render 账户
访问 https://render.com 注册账户（支持 GitHub 登录）

### 2. 创建新 Web Service
1. 点击 "New" → "Web Service"
2. 连接你的 GitHub 仓库（需要先将代码推送到 GitHub）
3. 或者选择 "Public Git repository" 并输入仓库地址

### 3. 配置部署
- **Name**: xinling-app
- **Environment**: Docker
- **Branch**: main (或你的主分支)
- **Root Directory**: 亲和之心-(sanctuary) (如果代码在子目录)

### 4. 设置环境变量
在 Environment 部分添加：
```
AI_API_KEY=你的DashScope_API密钥
AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen-turbo
NODE_ENV=production
```

### 5. 部署
点击 "Create Web Service" 开始部署

### 6. 获取访问链接
部署完成后，Render 会提供一个公网 URL（如：https://xinling-app.onrender.com）

## 🌐 备用方案：Railway 部署

### 1. 注册 Railway 账户
访问 https://railway.app 注册账户

### 2. 创建新项目
1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 连接你的 GitHub 仓库

### 3. 配置环境变量
```
AI_API_KEY=你的DashScope_API密钥
AI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen-turbo
NODE_ENV=production
```

### 4. 自动部署
Railway 会自动检测 Dockerfile 并部署

## 📋 部署前准备

### 1. 推送到 GitHub
```bash
# 创建 Git 仓库
git init
git add .
git commit -m "Initial commit"

# 创建 GitHub 仓库并推送
# 在 GitHub 上创建新仓库，然后：
git remote add origin https://github.com/yourusername/xinling-app.git
git push -u origin main
```

### 2. 获取 DashScope API 密钥
1. 访问 https://dashscope.aliyuncs.com/
2. 注册账户并获取 API Key
3. 复制 API Key 用于部署配置

## ⚠️ 注意事项

1. **免费额度限制**：
   - Render: 每月 750 小时免费
   - Railway: 512MB RAM + $5 免费额度

2. **冷启动**：免费服务可能有冷启动延迟

3. **API 费用**：DashScope API 调用会产生费用

4. **域名**：可以绑定自定义域名

## 🔧 本地测试

```bash
npm run dev

npm run build

```

2. 检查环境变量配置
3. 确认 API 密钥有效性
