# 情感陪护数字人系统
## 原型制作风格与功能闭环文档（老年友好版）

## 1. 文档目标
本文件用于指导产品原型制作（Figma/Axure/即时设计），确保两件事同时成立：
1. 功能介绍完整，可覆盖演示与开发沟通需求；  
2. 视觉与交互风格温馨、易懂、可操作，符合老年人使用习惯。  

## 2. 产品定位与体验关键词

### 2.1 产品定位
一个提供情感陪伴、心理支持和轻干预建议的网页系统，用户通过文字或语音与数字人交流，系统持续评估情绪状态并给出温和引导，形成“初评-干预-复评-总结”闭环。

### 2.2 体验关键词（用于原型审美统一）
- 温馨陪伴  
- 稳定可信  
- 操作简单  
- 字大清晰  
- 反馈明确  

## 3. 目标用户与设计原则

### 3.1 目标用户
- 60岁以上用户（视力、听力、操作速度可能下降）  
- 焦虑、失眠、孤独等情绪波动人群  
- 家属辅助使用者（偶尔代操作）  

### 3.2 老年友好设计原则
1. 关键按钮大、少、固定位置。  
2. 核心流程单路径，不让用户做复杂选择。  
3. 所有状态有明确提示（听、想、说、复评、完成）。  
4. 文案避免专业术语，避免压力表达。  
5. 优先容错：误触可撤回、失败可重试、无权限可降级。  

## 4. 风格规范（原型视觉基线）

## 4.1 色彩规范（温馨低刺激）
- 主色：`#6FAFA4`（温和青绿）  
- 主色深：`#4C8F85`（按钮按下态）  
- 背景：`#F7F6F2`（暖白）  
- 卡片：`#FFFFFF`  
- 文字主：`#2F3A3A`  
- 文字次：`#6E7A7A`  
- 分割线：`#E6E9E9`  
- 成功：`#62B46E`  
- 警示：`#E6A23C`  
- 风险高：`#E87070`  

风险颜色只用于风险卡片与提醒，不大面积铺色，避免制造焦虑。

## 4.2 字体与字号
- 字体：系统无衬线（优先苹方、HarmonyOS Sans或同类）
- 一级标题：28px / 半粗
- 二级标题：22px / 半粗
- 正文：18px / 常规
- 次要信息：16px / 常规
- 小字最小不低于14px

## 4.3 圆角、阴影与空间
- 卡片圆角：12px  
- 按钮圆角：10px  
- 控件高度：普通48px；主按钮56px  
- 阴影：轻阴影，避免“悬浮感过重”  
- 页面留白：模块间距至少16px，主区域至少24px  

## 4.4 图标与插画风格
- 线性圆角图标，线宽2px  
- 避免尖锐、恐慌、医疗压迫感图形  
- 可用轻度治愈插画（如暖色背景、室内陪伴场景）

## 5. 语气与文案规范（必须执行）

### 5.1 语气原则
- 先共情，再建议  
- 用“我们一起”替代“你应该”  
- 禁止命令式、训诫式表达  
- 避免过度承诺（如“我一定治好你”）

### 5.2 文案示例
- 欢迎语：  
  “您好，今天我会一直陪着您。我们可以慢慢聊，您想到什么就说什么。”

- 识别失败：  
  “我刚刚没有听清楚，您可以再说一遍，或者直接输入文字。”

- 复评完成：  
  “我帮您更新了当前状态，我们继续慢慢聊，已经比刚开始更稳定一些了。”

- 高风险提醒：  
  “您现在的状态让我有些担心。建议尽快联系家人或专业人员，我也可以先陪您做一次呼吸放松。”

## 6. 信息架构（IA）

1. 登录页  
2. 会话主页（核心）  
3. 会话总结页  
4. 历史记录页（可选）  

原型优先级：登录页 + 会话主页 + 总结页必须完整，高保真优先做会话主页。

## 7. 页面与功能完整说明（原型制作核心）

## 7.1 登录页（简洁、安全、一步进入）

### 页面目标
让用户快速进入系统，完成基础授权，不被复杂表单阻碍。

### 页面元素
1. 顶部：产品名称 + 温馨副标题  
2. 中部：昵称输入框、进入按钮  
3. 底部：隐私说明简述（简洁一行）  
4. 首次进入权限弹窗：麦克风授权、摄像头可选授权  

### 交互规则
- 默认只要求昵称，其他信息不必填。  
- 若用户拒绝麦克风，系统自动切到文字模式，不阻断使用。  
- 进入按钮始终可见，避免滚动后找不到入口。  

---

## 7.2 会话主页（最核心原型页，需高保真）

### 页面目标
以“陪伴感+安全感”为核心，让用户看得懂、点得准、等得住。

### 布局建议（桌面端）
- 左侧 58%：数字人舞台区  
- 中间 27%：聊天消息区  
- 右侧 15%：风险与状态区  
- 底部固定：输入与控制区

### 模块A：数字人舞台区
**功能完整说明：**
1. 展示单数字人形象（正面半身）  
2. 状态标签：聆听中 / 思考中 / 回应中 / 复评中  
3. 播报时口型联动  
4. 表情联动（平静、关怀、鼓励、担忧、肯定）

**原型细节：**
- 状态标签固定在数字人下方，颜色柔和。  
- 播报时加轻微“声波光圈”效果，提示正在说话。  
- 思考中不使用“加载菊花”，改用“我正在整理您的情况…”文字。  

### 模块B：聊天消息区
**功能完整说明：**
1. 展示用户消息和数字人回复  
2. 支持流式回复（逐字出现）  
3. 支持关键回复一键“再说一遍”  
4. 失败消息可重试

**原型细节：**
- 用户消息靠右，数字人消息靠左。  
- 每条消息显示时间（hh:mm）。  
- 数字人回复可附“建议卡片”（如呼吸训练、作息建议）。  

### 模块C：风险与状态区
**功能完整说明：**
1. 三类风险卡：焦虑 / 抑郁 / 双相  
2. 每类展示分值（0-100）与等级（低/中/高）  
3. 展示复评变化（上升/下降/持平）  
4. 展示当前策略（安抚陪伴 / 引导干预 / 转介提醒）

**原型细节：**
- 风险卡使用小面积色条，不要整卡红色。  
- 变化箭头旁配文：  
  - 上升：需要更多关注  
  - 下降：状态在改善  
  - 持平：保持当前节奏  

### 模块D：输入与控制区（底部固定）
**功能完整说明：**
1. 文本输入框（大尺寸）  
2. 语音输入按钮（开始/结束）  
3. 发送按钮  
4. 停止播报按钮  
5. 结束会话按钮（需二次确认）

**原型细节：**
- 主操作按钮至少56px高。  
- 语音按钮必须有明显状态变化：灰（未启用）/绿（录音中）。  
- 结束会话按钮使用次级色，防误触。

## 7.3 会话总结页（闭环结果展示页）

### 页面目标
清晰告诉用户“本次聊了什么、状态如何变化、接下来怎么做”。

### 页面模块
1. 会话概览：时长、轮次、主要话题  
2. 风险变化：初评->复评趋势  
3. 干预建议：3-5条可执行建议  
4. 结束提醒：如有必要提示联系家人或专业支持  
5. 操作：复制摘要、开始新会话

### 文案风格
- 正向鼓励 + 可执行建议  
- 避免“严重”“危险”等刺激词，改为“需要更多关注”

## 8. 闭环功能设计（必须完整呈现）

本系统最重要的是“闭环”，原型中必须明确以下四步：

### 8.1 初评（第一阶段）
- 用户前1-2轮输入后，系统生成初始风险评估。  
- 在右侧风险区出现“首次评估完成”状态提示。  

### 8.2 干预（第二阶段）
- 数字人按风险等级执行不同策略：
  - 低风险：倾听+轻建议  
  - 中风险：结构化引导+行为建议  
  - 高风险：稳态安抚+转介提醒  
- 每条干预建议以卡片形式可视化，避免“只说不记”。  

### 8.3 复评（第三阶段）
- 每3-5轮自动复评一次。  
- 若风险降低，界面给出“状态改善”；  
- 若风险升高，自动提高策略等级并提示更多支持建议。  

### 8.4 总结（第四阶段）
- 结束会话时自动生成总结：  
  1) 当前状态  
  2) 变化趋势  
  3) 下一步建议  
- 用户可复制总结给家属或保存。  

## 9. 关键状态与异常闭环（原型必须画出）

### 9.1 状态闭环
- `待输入` -> `聆听中` -> `处理中` -> `播报中` -> `复评中` -> `待输入`

### 9.2 异常闭环
1. ASR失败 -> 提示重说/改文本 -> 回到待输入  
2. LLM超时 -> 提示稍后重试 -> 一键重试  
3. TTS失败 -> 自动文本回退 -> 不中断会话  
4. 网络中断 -> 顶部警示条 -> 点击重连 -> 恢复会话  

## 10. 无障碍与老年可用性要求

1. 支持页面缩放到125%-150%仍不乱版。  
2. 支持键盘Tab可达关键按钮。  
3. 文案对比度满足可读性（主文字与背景对比明显）。  
4. 关键操作必须有二次确认（结束会话、清空记录）。  
5. 语音按钮、发送按钮至少44x44px（建议56x56）。  

## 11. 原型标注规范（交付给开发）

每个页面需标注：
1. 组件名称  
2. 交互触发条件  
3. 状态变化  
4. 接口字段（输入/输出）  
5. 异常兜底行为  

建议在原型中使用统一标注前缀：
- `[UI]` 视觉说明  
- `[INT]` 交互说明  
- `[API]` 接口说明  
- `[ERR]` 异常说明  

## 12. 核心功能清单（完整版）

1. 登录与基础授权  
2. 文本输入与发送  
3. 语音输入与识别  
4. 数字人回复与播报  
5. 口型与表情联动  
6. 风险评估展示（三分类）  
7. 自动复评与策略切换  
8. 会话总结生成  
9. 历史会话查看（可选）  
10. 错误提示与恢复机制  

## 13. 原型验收标准

1. 用户可在3步内开始第一次对话。  
2. 主要按钮均可一眼识别，误触率低。  
3. 会话中每个系统状态均有清晰反馈。  
4. 风险评估与复评变化可视可理解。  
5. 总结页可明确表达“从哪里到哪里、下一步怎么做”。  
6. 整体风格温馨克制，不产生焦虑感。  

## 14. 演示脚本（用于原型走查）

1. 打开登录页，输入昵称进入。  
2. 授权麦克风，数字人欢迎。  
3. 用户说“最近晚上总睡不好”。  
4. 数字人共情回应并给出第一个建议。  
5. 第4轮触发复评，风险卡显示“下降”。  
6. 点击结束会话，进入总结页。  
7. 展示建议列表并复制摘要。
