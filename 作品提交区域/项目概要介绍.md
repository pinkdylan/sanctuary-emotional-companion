# 情感陪护数字人系统 Web 产品方案（PRD）

## 1. 文档信息
- 产品名称：情感陪护虚拟数字人（Web端）
- 文档版本：v1.0（MVP）
- 目标读者：前端开发、后端接口开发、测试、演示人员
- 产品目标：在网页端实现“感知-认知-干预-复评”最小闭环，支持1个数字人角色完成稳定的情感陪护对话

## 2. 目标与范围

### 2.1 目标
1. 支持文本+语音双通道交互。  
2. 提供单数字人实时播报（TTS）+口型+表情联动展示。  
3. 展示风险评估结果（焦虑/抑郁/双相）及复评变化。  
4. 提供完整会话记录与会话总结导出。  

### 2.2 MVP范围
**包含：**
- 登录（可简化为演示码登录）
- 会话主页（数字人区、对话区、风险区、控制区）
- 实时文本流式回复
- 语音输入、语音播报
- 风险卡片与趋势简图
- 会话结束总结页

**不包含（后续）：**
- 多角色切换
- 复杂3D数字人
- 多用户协同管理后台

## 3. 用户与使用场景

### 3.1 核心用户
- 被陪护用户：老年人、慢病康复者、独居人群
- 操作辅助者：家属/护工（帮忙打开网页和设备）

### 3.2 典型场景
1. 用户说“最近总睡不好，很焦虑”。  
2. 系统识别情绪与风险，数字人共情回应并给出引导。  
3. 对话进行3-5轮后自动复评。  
4. 会话结束输出总结建议（如呼吸训练、作息建议、必要时就医提醒）。  

## 4. 信息架构（IA）

1. `登录页`  
2. `会话页（主页面）`  
   - 数字人展示区
   - 对话消息区
   - 输入与控制区
   - 风险与状态侧栏
3. `会话总结页`  
4. `历史记录页（可选MVP简版）`  

## 5. 页面详细需求

## 5.1 登录页
### 功能
- 输入昵称/演示码后进入系统
- 设备授权引导（麦克风、摄像头可选）

### 关键交互
- 首次进入弹出权限引导弹窗
- 麦克风拒绝授权时，自动降级为文本模式并提示

### 字段
- `nickname`（必填）
- `demoCode`（选填）

---

## 5.2 会话页（核心页面）

### 5.2.1 布局（建议）
- 左侧（60%）：数字人视频画布 + 状态提示（聆听中/思考中/播报中）
- 中间（25%）：聊天消息流
- 右侧（15%）：风险卡片、复评状态、系统提示
- 底部：文本输入框 + 语音按钮 + 结束会话按钮

### 5.2.2 功能模块
1. 数字人展示模块  
   - 播报时触发口型动画
   - 根据回复情绪标签切换表情（平静/关怀/鼓励/担忧/肯定）

2. 对话模块  
   - 用户文本发送
   - 语音输入（按住说话或点击开始/结束）
   - AI流式回复（打字机效果）
   - 消息状态：发送中/已送达/失败重试

3. 风险评估模块  
   - 展示三类风险分值（0-100）与等级（低/中/高）
   - 显示“本轮与上一轮变化”箭头（上升/下降/持平）

4. 复评模块  
   - 每3-5轮自动触发复评
   - 复评时在系统消息中插入“复评完成，策略已更新”

5. 会话控制模块  
   - 暂停播报
   - 重新生成本轮回复
   - 结束会话并生成总结

### 5.2.3 状态机（前端）
- `idle`：待输入  
- `listening`：语音采集中  
- `processing`：ASR/LLM处理中  
- `speaking`：TTS播报中  
- `reassessing`：复评中  
- `error`：异常状态  

### 5.2.4 异常处理
- ASR失败：提示“未识别清楚，请重试或改用文本输入”
- LLM超时：提示“当前响应较慢，请稍后或重试”
- TTS失败：自动回退仅文本展示
- 网络断开：消息区顶部红条提示，可手动重连

---

## 5.3 会话总结页
### 功能
- 展示会话概览（时长、轮次、关键词）
- 展示风险变化曲线（初评 vs 复评）
- 展示干预建议摘要（3-5条）
- 支持导出TXT/复制摘要

### 内容结构
1. 会话基础信息
2. 风险趋势
3. 关键对话片段
4. 后续建议与注意事项

---

## 5.4 历史记录页（MVP简版）
### 功能
- 展示最近N次会话列表
- 支持查看单次总结

### 列表字段
- 会话ID
- 日期时间
- 会话时长
- 风险最高等级
- 查看详情按钮

## 6. 核心交互流程

### 6.1 首次使用流程
1. 用户登录  
2. 授权麦克风  
3. 进入会话页，系统欢迎语播报  
4. 用户开始文本或语音输入  

### 6.2 对话与复评流程
1. 用户输入 -> 前端发送消息  
2. 展示“处理中”状态  
3. 接收流式文本并实时渲染  
4. 同步触发TTS与数字人口型  
5. 达到复评轮次后更新风险卡片和策略提示  

### 6.3 会话结束流程
1. 点击“结束会话”  
2. 调用总结接口  
3. 跳转总结页并展示建议  

## 7. 视觉与UI规范（前端可直接执行）

### 7.1 设计基调
- 关键词：温和、可信、安静、陪伴感
- 主色：低饱和蓝绿（如 `#5AA9A4`）
- 辅色：中性灰、浅米白背景
- 风险色：
  - 低：绿色
  - 中：橙色
  - 高：红色

### 7.2 字体与排版
- 中文默认：系统无衬线字体
- 正文：14-16px
- 标题：20-28px
- 行高：1.5-1.7

### 7.3 组件风格
- 圆角卡片（8-12px）
- 阴影轻量
- 大按钮（方便老年用户点击）
- 重要按钮需二次确认（结束会话）

## 8. 前端组件清单

1. `AvatarStage`：数字人舞台组件（口型/表情/状态）  
2. `ChatMessageList`：消息流组件  
3. `ChatInputBar`：文本输入+发送  
4. `VoiceInputButton`：语音采集控件  
5. `RiskPanel`：风险分值卡片  
6. `SessionStatusBadge`：状态提示条  
7. `SummaryCard`：总结信息卡片  
8. `PermissionModal`：权限引导弹窗  
9. `Toast/Alert`：错误提示组件  

## 9. 接口约定（MVP建议）

## 9.1 登录
- `POST /api/auth/login`
- req:
```json
{
  "nickname": "用户A",
  "demoCode": "optional"
}
```
- resp:
```json
{
  "token": "xxx",
  "userId": "u_001"
}
```

## 9.2 发送消息（文本）
- `POST /api/chat/message`
- req:
```json
{
  "sessionId": "s_001",
  "content": "我最近睡不好",
  "inputType": "text"
}
```
- resp（流式/分段）:
```json
{
  "replyText": "听起来你最近很辛苦...",
  "emotionTag": "care",
  "risk": {
    "anxiety": 68,
    "depression": 42,
    "bipolar": 21,
    "level": "medium"
  },
  "reassessTriggered": false
}
```

## 9.3 语音上传识别
- `POST /api/asr/recognize`
- form-data: `audioFile`
- resp:
```json
{
  "text": "最近总是心慌",
  "confidence": 0.91
}
```

## 9.4 文本转语音
- `POST /api/tts/synthesize`
- req:
```json
{
  "text": "我理解你的感受，我们可以慢慢来。",
  "voiceStyle": "warm"
}
```
- resp:
```json
{
  "audioUrl": "https://xxx/audio/001.wav",
  "phonemeTimeline": [
    {"p":"wo","t":0.02},
    {"p":"li","t":0.10}
  ]
}
```

## 9.5 会话总结
- `POST /api/session/summary`
- req:
```json
{
  "sessionId": "s_001"
}
```
- resp:
```json
{
  "durationSec": 780,
  "turns": 14,
  "riskTrend": {
    "anxiety": [72,65,58],
    "depression": [45,43,39],
    "bipolar": [22,21,21]
  },
  "suggestions": [
    "今天可先尝试5分钟呼吸训练",
    "建议固定睡前放松流程",
    "若持续不适请寻求专业帮助"
  ]
}
```

## 10. 埋点与数据统计（前端）
- 页面访问：`page_view`
- 首次授权成功率：`mic_permission_granted`
- 消息发送成功率：`chat_send_success`
- 平均响应时长：`chat_latency_ms`
- 会话完成率：`session_completed`
- 复评触发率：`reassess_triggered`

## 11. 非功能需求
- 兼容：Chrome/Edge 最新两个版本
- 响应性能：首屏 <= 3s（局域网演示环境）
- 可用性：关键路径错误可恢复
- 隐私：会话数据脱敏存储，日志不落明文敏感信息

## 12. 验收标准（前端向）
1. 文本+语音输入均可用  
2. 单数字人播报时口型和表情正常联动  
3. 连续10轮对话界面稳定无崩溃  
4. 风险卡片可实时更新并显示复评变化  
5. 会话结束可生成总结并可复制导出  
6. 异常提示明确且可恢复操作  

## 13. 开发任务拆分（前端）

### Sprint 1（3-4天）
- 完成登录页、会话页基础布局、消息流组件
- 对接文本聊天接口（先mock）

### Sprint 2（3-4天）
- 接入语音输入、流式回复、TTS播放
- 完成数字人舞台组件（口型与状态联动）

### Sprint 3（3-4天）
- 接入风险面板、复评提示、总结页
- 完成异常处理、埋点、联调与验收
