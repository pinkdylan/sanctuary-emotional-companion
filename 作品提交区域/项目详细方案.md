# 基于AI大语言模型的情感陪护虚拟数字人系统
## 项目详细方案（MVP闭环版）

## 1. 项目概述

### 1.1 项目背景
面向空巢老人、慢病康复人群及有情感支持需求的用户，构建基于大语言模型与数字人交互技术的主动式心理陪护系统。系统围绕“感知-认知-干预-再评估”形成业务闭环，提供可持续、可量化的情感支持服务。

### 1.2 项目目标
1. 实现文本/语音双通道自然交互，支持连续多轮对话。  
2. 构建1个2D数字人形象，实现语音播报、口型同步、基础表情与动作。  
3. 完成多模态信息采集与融合，输出焦虑、抑郁、双向情感障碍风险倾向评估。  
4. 接入心理学知识库，生成共情化、专业化、非模板化干预对话。  
5. 完成一次会话内“初评-干预-复评”的主动闭环，并沉淀可追踪报告。  
6. 满足项目技术要求并输出可执行工程交付件。

### 1.3 建设原则
- 最小可行：优先保证闭环跑通与核心指标达成。  
- 安全可信：建立风险分级与敏感内容安全护栏。  
- 可扩展：模块化设计，支持后续3D形象和更强融合模型升级。  

## 2. 需求分析与范围界定

### 2.1 对照任务书的核心需求
1. 交互与服务：数字人形象、TTS、Lip-sync、文本和语音交互、低延迟。  
2. 多模态采集：摄像头/麦克风采集用于心理状态分析。  
3. 智能与驱动：LLM+心理知识库，输出共情和专业内容。  
4. 多模态融合：视频、语音、文本的时空对齐与语义融合。  
5. 面部行为驱动：实现音画同步、表情动作生成与评估。  

### 2.2 MVP范围（覆盖至少80%要求）
**纳入MVP：**
- 双模态交互（文本+语音）  
- 1个2D数字人（默认温和陪伴风格）  
- 实时TTS+Lip-sync+基础表情（平静、关怀、鼓励、担忧、肯定）  
- 音频和文本必选采集，视频模态实现采集接口及基础特征接入  
- 焦虑/抑郁/双相风险三分类倾向评估（低/中/高）  
- 10轮以上连续对话，上下文感知、追问、澄清、总结  
- 会话内复评机制（每3-5轮更新风险并调整策略）  
- Docker化部署与可执行模型工程交付

**暂缓项（后续增强）：**
- 高保真3D角色与复杂肢体动作  
- 端到端重模型视频行为识别  
- 长周期个体画像与跨会话高级记忆  

## 3. 总体技术方案

### 3.1 系统总体架构
系统采用“前端交互层 + 实时服务层 + 智能决策层 + 数据与安全层”的分层架构。

1. 前端交互层  
   - Web交互界面（文本输入、语音按钮、会话展示）  
   - 数字人渲染与驱动（2D角色、口型、表情、动作）

2. 实时服务层  
   - ASR语音识别服务  
   - TTS语音合成服务  
   - 实时会话编排与状态机服务

3. 智能决策层  
   - LLM对话生成模块  
   - 心理知识库检索增强（RAG）  
   - 多模态风险评估与融合模块  
   - 干预策略选择器（低/中/高风险分级响应）

4. 数据与安全层  
   - 会话日志、风险轨迹、干预记录  
   - 内容安全策略（敏感词、危机话术、升级建议）

### 3.2 关键流程（闭环）
1. 用户输入（语音/文本）  
2. ASR转写 + 语音情感特征提取 + 文本语义特征提取  
3. 多模态融合输出风险等级及置信度  
4. LLM结合知识库生成干预内容  
5. 输出文本与语音，驱动数字人口型与表情  
6. 每N轮触发复评，更新风险并动态调整策略  
7. 会话结束输出总结和后续建议

## 4. 模块设计

### 4.1 数字人交互模块
- 提供1个2D数字人形象，支持语音驱动口型与表情联动。  
- 音频驱动口型同步，支持基础面部表情及轻量动作。  
- 目标体验：语音播报自然，视听同步，交互不中断。

### 4.2 语音识别模块（ASR）
- 输入：用户麦克风流式音频。  
- 输出：带时间戳转写文本。  
- 目标指标：WER<=10%，SER<=40%（通过数据增强与热词优化达成）。

### 4.3 大语言模型对话模块
- 输入：转写文本 + 会话历史 + 风险评估结果 + 检索知识片段。  
- 输出：非模板化、可追问、可澄清、可总结的干预文本。  
- 能力要求：支持>=10轮上下文，窗口长度>=8k tokens，单样本响应<=60秒。

### 4.4 心理知识库与RAG模块
- 知识内容：情绪疏导、睡眠干预、焦虑缓解、危机识别建议、沟通禁忌。  
- 检索策略：按风险类型和意图进行Top-K检索，再融合到Prompt。  
- 作用：降低幻觉，提高专业性和一致性。

### 4.5 多模态融合评估模块
- 文本特征：情绪词、否定倾向、绝望语义、风险关键词。  
- 语音特征：语速、音强、停顿、情绪概率。  
- 视频特征（MVP）：基础面部特征接口接入，可选启用。  
- 融合输出：焦虑/抑郁/双相风险倾向分值与分级结果。

### 4.6 干预与再评估模块
- 按风险等级选择对话策略：  
  - 低风险：倾听、情绪确认、轻度建议；  
  - 中风险：结构化引导、行为建议、复述确认；  
  - 高风险：稳态安抚、明确转介建议、危机提示。  
- 每3-5轮触发复评，若风险升高则自动切换更稳妥策略。

### 4.7 安全策略模块
- 内容安全：屏蔽不当建议、避免医学诊断结论式表达。  
- 危机规则：出现自伤/他伤倾向关键词时触发高优先级处置模板。  
- 输出边界：强调“心理支持”而非“医疗诊断替代”。

## 5. 关键技术路线与实现要点

### 5.1 交互低延迟策略
- 流式ASR与流式TTS并行。  
- LLM采用分段响应（先安抚后详细建议）。  
- 前后端异步队列化，避免数字人渲染阻塞。

### 5.2 指标达成策略
1. ASR指标达成  
   - 场景化语料微调与噪声增强；  
   - 养老场景热词表（睡眠、血压、孤独、胸闷等）；  
   - VAD切分优化减少断句错误。  

2. LLM指标达成  
   - 选用8k+上下文模型；  
   - 采用会话记忆摘要减少上下文冗余；  
   - 将知识检索片段限制长度以控制响应时间。  

3. 数字人驱动指标提升  
   - 音素级口型映射+平滑滤波；  
   - 表情按情绪标签触发并加防抖机制；  
   - 单角色场景下优先保证同步性和真实性。  

### 5.3 可扩展设计
- 模块接口标准化（ASR/LLM/TTS/融合器均可替换）。  
- 视频模态从“可选接入”平滑升级到“核心融合”。  
- 2D角色可迁移到3D角色驱动管线。

## 6. 评估体系与测试方案

### 6.1 功能验收测试
1. 文本交互：连续10轮对话无中断。  
2. 语音交互：音频输入、识别、回复、播报全链路可用。  
3. 数字人驱动：口型随音频变化，表情与语义一致。  
4. 闭环机制：有初评、干预、复评与总结报告。  
5. 单角色验收：1个数字人可稳定驱动并完成全流程交互。  

### 6.2 性能测试
- 响应时延：单轮推理响应<=60秒。  
- 稳定性：30分钟连续会话无闪退。  
- 并发性：基础并发场景（2-5路）服务可用。

### 6.3 模型评估
- ASR：WER、SER指标按测试集评估。  
- 对话质量：共情性、连贯性、专业性人工评分。  
- 风险评估：与标注样本比对准确率/召回率。  
- 数字人驱动：按赛题指标框架执行评估与记录。  

## 7. 实施计划与里程碑

### 7.1 周期规划（4周）
**第1周：基础链路打通**  
- 完成前端交互页面与会话服务。  
- 接入ASR、LLM、TTS，形成文本/语音问答闭环。  
- 上线1个2D数字人基础播报与口型。

**第2周：认知能力上线**  
- 完成心理知识库整理与RAG检索。  
- 上线三类风险识别模型（文本+语音）。  
- 增加追问、澄清、总结策略模板。

**第3周：闭环强化**  
- 实现多模态融合评分与冲突处理规则。  
- 上线复评机制与策略动态切换。  
- 形成会话报告与风险趋势图。

**第4周：冲刺交付**  
- 指标调优（WER/SER、时延、稳定性）。  
- 完成Docker镜像与工程打包。  
- 录制演示视频，完成PPT与文档定稿。  

### 7.2 人员分工建议
- 算法负责人：ASR、风险识别、融合模型  
- 平台负责人：后端服务编排、数据与接口  
- 前端负责人：交互界面与数字人渲染  
- 测试与文档：测试用例、演示材料、提交材料整理  

## 8. 风险识别与应对

1. 语音识别精度不足  
   - 应对：数据增强+热词+分场景降噪；先保障安静室内达标。  

2. 对话质量波动  
   - 应对：RAG约束+系统提示词分层+失败重试。  

3. 数字人口型不同步  
   - 应对：音频缓存窗口与驱动帧平滑；优先保证同步性。  

4. 复评逻辑不稳定  
   - 应对：规则兜底（关键词触发+风险阈值防抖）。  

5. 交付时间紧张  
   - 应对：按MVP优先级冻结非核心功能，先保闭环演示和提交件。
