### 1. 项目背景与意义
#### 1.1 社会背景
- 我国60岁以上人口突破3亿，空巢老人占比超50%
- 老年群体面临"物质保障充足但精神孤独突出"的矛盾
- 传统养老服务在情感陪伴维度存在显著缺口
- 医疗机构日均接待陪诊需求超3000人次
- 养老机构中98.3%的失能老人存在持续性情感支持需求

#### 1.2 项目定位
- 构建以数字人为核心的"感知-认知-干预"主动闭环系统
- 实现对用户心理状态的评估、引导/干预、再评估的闭环
- 提供可持续、可陪伴、可引导的心理健康、情感支持服务

#### 1.3 技术挑战
- 多模态数据采集与融合（语音、文本、视频）
- 大语言模型与心理学知识库的深度集成
- 数字人面部行为驱动与音画同步
- 实时交互的低延迟要求

### 2. 需求分析与目标
#### 2.1 核心需求
1. **交互与服务**
   - 提供表现力丰富的2D/3D数字人形象
   - 支持自然流畅的语音对话（TTS）
   - 精准的口型同步（Lip-sync）
   - 丰富的面部表情和适度的肢体动作
   - 支持文本和语音两种主流交互模式

2. **智能与驱动**
   - 基于LLM与心理学知识库开发专用AI模型
   - 生成富有共情力和专业性的对话内容
   - 对视频、语音、文本特征进行时空对齐与语义融合
   - 构建数字人面部行为驱动模型

#### 2.2 技术指标要求
- **语音识别**：WER ≤ 10%，SER ≤ 40%
- **大语言模型**：上下文窗口 ≥ 8k tokens，响应时间 ≤ 60秒
- **面部行为驱动**：综合评分S计算（FRCorr/FRdist等7项指标）

#### 2.3 MVP范围界定
- **核心功能**：文本+语音双通道交互，单数字人实时播报，风险评估与复评
- **交付物**：Web应用、Docker镜像、可执行模型工程文件

#### 3.1 总体架构
采用"前端交互层 + 实时服务层 + 智能决策层 + 数据与安全层"的分层架构

   - 多模态风险评估与融合模块

### 4. 技术实现方案
#### 4.1 数字人交互模块
- **技术选型**：Live2D 2D数字人 + PixiJS渲染引擎
- **核心依赖**：
  - `pixi-live2d-display@^0.5.0-beta`：Live2D显示组件
  - `pixi.js@^7.4.3`：2D图形渲染引擎
- **功能实现**：
  - 语音驱动口型同步：基于拼音音节到viseme的映射算法
  - 基础表情系统：平静(calm)/关怀(care)/鼓励(encourage)/担忧(worry)/肯定(affirm)
  - 轻量动作支持：头部轻微摆动，眨眼动画
- **口型同步算法**：
  - 输入：中文文本 → 拼音分词 → 音节映射 → viseme序列
  - viseme映射：基于Oculus风格，支持AA/E/I/O/U/DD等发音状态
  - 时间轴对齐：按音节权重分配总时长，生成带时间戳的viseme序列
- **性能优化**：实时渲染，视听同步延迟<100ms

#### 4.2 语音识别模块（ASR）
- **当前实现**：模拟ASR服务（开发阶段）
- **接口设计**：`/api/asr/recognize` POST接口
- **技术方案**：预留第三方ASR API集成接口
- **目标指标**：WER ≤ 10%，SER ≤ 40%
- **数据增强策略**：
  - 热词优化：心理健康相关词汇优先识别
  - 噪声处理：环境噪音过滤算法
  - 多语言支持：中文普通话优化

#### 4.3 大语言模型对话模块
- **模型选择**：Dashscope Qwen-turbo
- **API集成**：兼容OpenAI格式的chat/completions接口
- **核心参数**：
  - 模型：qwen-turbo
  - 温度：0.7（平衡创造性和一致性）
  - 上下文窗口：≥8k tokens
- **能力要求**：
  - 支持 ≥ 10轮连续对话
  - 上下文感知：维护对话历史和用户状态
  - 非模板化回复：避免机械化回答
  - 追问响应：主动引导深入对话
  - 语义澄清：处理模糊表达
  - 内容总结：生成会话摘要
- **响应时间**：单样本推理 ≤ 60秒（实际测试30-45秒）
- **错误处理**：网络异常重试，内容过滤

#### 4.4 心理知识库与RAG模块
- **知识库结构**：JSON格式存储，包含标题、分类、关键词、内容
- **检索算法**：
  - 分词策略：中文分词 + 二元组提取
  - 匹配机制：关键词重叠 + 语义相似度
  - 排序算法：BM25-like评分系统
- **RAG集成**：
  - 检索增强：Top-K条目检索（默认K=5）
  - Prompt注入：将检索结果作为系统提示前缀
  - 幻觉控制：基于检索结果生成，避免虚构内容
- **知识内容分类**：
  - 情绪疏导：焦虑缓解、压力释放
  - 睡眠干预：入睡技巧、作息建议
  - 危机识别：自杀倾向识别与干预
  - 沟通技巧：有效倾听、共情表达

#### 4.5 风险评估模块
- **评估算法**：基于关键词模式匹配的轻量级评估
- **评估维度**：
  - 焦虑倾向：检测"睡不好"、"心慌"、"担心"等信号词
  - 抑郁倾向：检测"累"、"难受"、"没意思"等信号词
  - 双向情感障碍：检测情绪波动相关表达
- **分级标准**：
  - 低风险：0-45分
  - 中风险：45-70分
  - 高风险：70分以上
- **复评机制**：
  - 触发条件：每3-5轮对话自动触发
  - 策略调整：根据风险等级调整对话语气和干预强度
- **可视化组件**：
  - 风险卡片：实时显示三维度分值和等级
  - 趋势图表：使用ECharts展示历史变化
  - 颜色编码：绿/黄/红表示不同风险等级

#### 4.6 前后端架构技术栈
- **前端技术栈**：
  - 框架：React 19.0.0 + TypeScript 5.8.2
  - 构建工具：Vite 6.2.0 + @vitejs/plugin-react
  - 状态管理：Zustand 5.0.12
  - 路由：React Router DOM 7.13.2
  - UI组件：Tailwind CSS 4.1.14 + Lucide React图标
  - 动画：Framer Motion 12.38.0
  - 图表：ECharts 6.0.0 + echarts-for-react

- **后端技术栈**：
  - 运行时：Node.js 22 + TypeScript
  - Web框架：Express 4.21.2
  - 开发工具：tsx 4.21.0（支持TypeScript执行）
  - 环境配置：dotenv 17.2.3
  - 工具库：pinyin 4.0.0（中文拼音转换）

#### 4.7 外部服务集成
- **LLM服务**：阿里云Dashscope平台
  - API地址：https://dashscope.aliyuncs.com/compatible-mode/v1
  - 认证方式：Bearer Token
  - 费用控制：请求频率限制和错误重试

- **TTS服务**：CosyVoice FastAPI
  - 接口：/inference_sft
  - 音频格式：16-bit PCM单声道
  - 采样率：22050Hz
  - 音色：中文女声（可配置）

   ```bash
   git clone <repository-url>
   cd 亲和之心-(sanctuary)
   ```

3. **配置环境变量**（可选）：
   ```bash
   cp .env.example .env
   # 编辑.env文件，填入API密钥
   ```

4. **构建和运行**：
   ```bash
   # 使用Docker Compose（推荐）
   docker-compose up -d

   docker run -p 3000:3000 xinling-app
   ```
